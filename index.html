实验四 Web 站点构建

构建内容

  • 新增课程项目总览、小组分工、实验二、实验三和实验四文章。

站点结构

实验三 代码编辑器与 Markdown

实验三主要完成 VS Code 编辑器配置、Markdown 插件安装和 Markdown 文档编写预览。

VS Code 版本验证

修复 PATH 后，在 PowerShell 中执行 code --version，成功输出 VS Code 版本号。

Markdown 插件安装

在 VS Code 扩展面板中安装 Markdown All in One、Markdown Preview Enhanced、Paste Image 和 markdownlint。

Markdown 编辑

创建 test.md，练习标题、强调、列表、代码块和表格等基础语法。

Markdown 预览

使用 VS Code 的 Markdown 预览功能，对照查看源码与渲染效果。

小结

实验三完成了代码编辑器和 Markdown 写作环境配置，为后续实验报告编写和站点内容维护提供了工具基础。

实验二 Web 软件开发环境

实验二主要完成 Web 开发环境搭建，包括 Node.js、npm、Git 和 Hexo CLI 的安装与验证，并初始化一个 Hexo 静态博客项目。

环境验证

通过命令行检查 Node.js、npm、Git 和 Hexo CLI 的版本，确认基础开发环境可用。

安装依赖

进入 Hexo 项目目录后执行 npm install，安装项目依赖。安装过程中出现少量 deprecated 警告，但最终依赖安装成功。

生成静态文件

执行 hexo generate，Hexo 成功生成首页、归档页、CSS 和 JavaScript 文件。

启动本地服务

执行 hexo server，本地服务运行在 http://localhost:4000/。

浏览器访问

在浏览器中访问本地地址，成功显示 Hexo 默认首页。

小结

实验二完成了 Web 开发环境搭建，为后续站点构建和部署提供了基础。

小组分工说明

本项目由 6 人小组共同完成。为了让任务清晰、成果可追踪，小组按照“环境、文档、Linux、远程管理、部署、整合”进行分工。

成员 角色 主要任务 产出
成员A 组长/集成负责人 统筹进度、检查材料、整合报告 总报告、提交包、README
成员B Web 环境与 Hexo 负责人 Node.js、npm、Git、Hexo、站点构建 实验二/四材料
成员C 文档与 VS Code 负责人 VS Code、Markdown、插件、截图整理 实验三材料
成员D Linux/虚拟机负责人 Ubuntu、Linux 命令、环境变量 实验五/六材料
成员E SSH/远程管理负责人 SSH 服务、密钥登录、文件传输 实验七材料
成员F Git/部署负责人 Git 仓库、GitHub Pages/Nginx 部署 实验八材料

协作方式

  • 使用 GitHub 仓库统一管理源码。
  • 每次修改后提交清晰的 commit message。
  • 截图按实验编号分类保存。
  • 最终由组长统一整合 PDF 报告和源码包。

课程项目

软件开发工具实践课程项目总览

本项目是软件开发工具实践短学期课程的小组成果展示站点。课程要求以 5-7 人小组为单位，完成一个 Web 静态站点从本地构建到远程部署的全过程，并将实验过程总结作为网站内容进行展示。

项目目标

  • 搭建 Web 开发环境，掌握 Node.js、npm、Git 和 Hexo 的基础使用。
  • 使用 VS Code 和 Markdown 完成实验文档编写与预览。
  • 使用 Git 进行项目版本控制，并提交到远程仓库。
  • 使用 Ubuntu/Linux 环境完成用户、权限、环境变量和远程管理配置。
  • 使用 SSH、SCP、rsync 等工具完成远程登录和文件传输。
  • 使用 GitHub Pages 或 Nginx 完成静态站点部署。
